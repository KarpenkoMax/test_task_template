# Часть 2 — API ingest + UI (LLM разрешён)

В этой части можно пользоваться LLM, но нам важно посмотреть **как вы думаете**: какие допущения делаете, как выбираете компромиссы, где закладываетесь на нагрузку.

Полезные команды (без venv/npm на хосте):
```bash
docker compose up -d --build
docker compose exec backend pytest
docker compose exec frontend npm test
```

## Backend: ingestion под нагрузку

Нужно реализовать api эндпоинт для записи событий пикселя

Файлы:
- `backend/pixel/views.py` (endpoint `POST /api/ingest/`)
- `backend/pixel/serializers.py`

Примечание:
- В этом шаблоне ingest сделан на DRF `APIView` и обработчик `post` **синхронный**.
  Если хотите обсудить асинхронную обработку под нагрузку — расскажите, как бы вы это делали

Сейчас ingest не имплементирован.

Что хотим в целом:
- Принимать события от пикселя
- Задуматься о нагрузке
- Внятно обработать ошибки/валидацию (как именно — на ваше усмотрение)
