# Часть 2 (35–40 мин) — Ingest API + небольшой UI (LLM разрешён)

Цель: реализовать ingest‑эндпоинт (принимает запросы пикселя) и завершить небольшой UI «кабинета событий».

## Бэкенд (Django REST)

Работать в:
- `backend/pixel/views.py` (`ingest`)
- опционально `backend/pixel/serializers.py`

Эндпоинт:
- `POST /api/ingest/`

Что важно:
- Валидация и защита от плохих данных.
- Дизайн с учётом нагрузки: избегайте очевидно неэффективных паттернов.

Запуск тестов бэкенда:
```bash
cd backend
python manage.py test
```

## Фронтенд (React)

Работать в:
- `frontend/src/pages/EventsPage.tsx`
- `frontend/src/components/EventDetails.tsx`

Задачи по UI:
1) Подключить фильтры (`client_id`, `session_id`, `event_type`) к GET‑запросу.
2) Реализовать выбор строки и вывод деталей через отдельный компонент (например, `EventDetails`), продумав передачу данных и состояние.

Запуск:
```bash
cd frontend
npm run dev
```

## Примечания
- LLM разрешён в этой части, но с объяснениями решений.
- Мы оцениваем рассуждения и компромиссы, а не объём кода.
