# Часть 1 — Pixel JS (без LLM)

Пиксель уже используется в UI: при вводе в фильтры на странице Events вызывается Pixel.track(...).
Ваша задача — сделать доставку надёжной и безопасной для страницы.

Где работать:
- `frontend/src/pixel/pixel.ts`
- тесты: `frontend/src/pixel/pixel.test.ts`

Запуск тестов локально:
```bash
cd frontend
npm test
```

Запуск тестов через Docker Compose (без установки npm на хосте):
```bash
docker compose up -d --build
docker compose exec frontend npm test
```

## Требования

Нужно сделать так, чтобы проходили тесты из `pixel.test.ts`:

1) Если сетевой запрос падает (исключение), то `track()`:
   - **не выбрасывает исключение наружу**
   - кладёт payload события в очередь в `sessionStorage` (ключ `storageKey`)

2) При `init()` регистрируется обработчик `pagehide`, который:
   - читает очередь из `sessionStorage`
   - пытается отправить её через `navigator.sendBeacon` (если доступен)
   - при успехе очищает очередь
